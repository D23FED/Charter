Text

A single set of <span> can contain multiple paragraphs. The paragrahs are to be separated using their own sets of <p> tags. Please refrain from using <br> unless you need a very specific kind of carriage return (use sparingly). The class name given to a <span> will change the text styling as follows:

Phone Number links

Modern browsers will initiate the User's default communication application/methods. Mobile devices will recognize this as a phone number and initiate a call. The actual number dialed is indicated in the tel: attribute, not the rendered text. Thereore, the rendered text could say just about anything, which for accessibility reasons is generally not a good practice. To keep things accessible, it is best to use only the phone number (formatted however you like). Rendering the phone number itself also allows the User to read the number, dial the number manually or copy/paste the number into a text message or other application.

For example:

<p><strong>(Good Practice):</strong> Please call us today at: <a href="tel:+18885551010">1-888-555-1010</a>. Operators are standing by.</p>
<p><strong>(Bad Practice):</strong> But don't call <a href="tel:+18885551010">this number</a>, because operators are not standing by.</p>

Will produce:

(Good Practice) Please call us today at: 1-888-555-1010! Operators are standing by.

(Bad Practice) But don't call this number, because operators are not standing by.

Form Elements

This section assumes input fields are already wrapped in the appropriate tags like <form>, <fieldset>, etc.

By default, many of the text input tags will extend the full width of the parent container. Those widths can be modified based on the grid being used and should not be declared individually or though any inline CSS.

<form>
   <div class="form-group">
      <input type="email" class="form-control" placeholder="Email">
      <input type="text" class="form-control" placeholder="First Name">
   </div>
</form>

Footer Navigation and Copyright Notice

To maintain a proper responsive grid, we are using classes from Bootstrap. The numbers in the column class (i.e.: col-md-4), should add up to 12 in order to stay in the same horizontal row. In this example, the copyright notice has the class col-md-12, meaning it will start a new row and span all the way accross.

<footer class="container-fluid">
   <div class="row-no-padding col-md-4">  <!-- <<< This number should add up to a max of 12 to keep in the same row. -->
      <ul>
         <li>Home</li>
         <li>My Account</li>
         <li>Locations</li>
         <li>Contact Us</li>
         <li>Services in My Area</li>
         <li>Business</li>
      </ul>
      <h6>Charter Spectrum Bundles</h6>
      <ul>
         <li>Spectrum Triple Play Select</li>
         <li>Spectrum Triple Play Silver</li>
         <li>Spectrum Triple Play Gold</li>
      </ul>
   </div>
   <div class="row-no-padding col-md-2">  <!-- <<< Since 4 + 2 = 6, this will fit in the same row... -->
      <h6>Products</h6>
      <ul>
         <li>Spectrum TV</li>
         <li>Spectrum Internet</li>
         <li>Spectrum Voice</li>
      </ul>
      <h6>Spectrum TV</h6>
      <ul>
         <li>HD & DVR</li>
         <li>ON Demand</li>
         <li>Spectrum TV App</li>
         <li>Genres</li>
         <li>Premium Channels</li>
         <li>Channel Lineup</li>
         <li>Why Charter?</li>
      </ul>
   </div>
   <div class="row-no-padding col-md-2">  <!-- <<< Since 6 + 2 = 8, this will fit in the same row... -->
      <h6>...[next title]</h6>
      <ul>
         <li>...[wash]</li>
         <li>...[rinse]</li>
         <li>...[repeat]</li>
      </ul>
      <h6>...[next title]</h6>
      <ul>
         <li>...[so on]</li>
         <li>...[so forth]</li>
      </ul>
   </div>
   .
   .
   .
   <div class="row-no-padding col-md-12">  <!-- <<< By now, the lists have exceeded 12. This starts a new row and spans accross. -->
      <p>© 2016 Charter Communications, Inc.</p>
   </div>
</footer>
